Infrastructure: Google Cloud Platform

Overview

Our pharma ordering platform requires several components: a website for customers, a database to store orders and products, a system to handle background tasks, and automated functions that respond to user input. These components need to work together reliably and securely.

While JAMP's current architecture is with Azure (Microsoft), I am most experienced and comfortable with Google Cloud. Since IT doesn't have the capacity to contribute to development on this project, learning an alternative platform would only increase development time.

Why Google Cloud

Integrated ecosystem: All services are pre-configured to communicate with each other without custom integration work. The website automatically connects to the database. No additional configuration required. This removes weeks of setup time that would otherwise be spent on networking between services.

Faster time to market: Less infrastructure complexity means more development time focused on business features rather than technical plumbing.

Trade-offs

Platform commitment: The architecture will use modular, portable components. If migration becomes necessary, the core application code remains unchanged; only infrastructure configuration needs updating. We are not stuck with our initial platform choice forever.

Recommendation

Use Google Cloud. The development time will be significantly lessened and it will be easier to train users on how to maintain / edit the platform moving forward. Creating documentation will also be easier.

What I need

IT to create a JAMP Google Cloud account as admin and add me as a member to build the project. No payment method needed immediately since the account comes with free credits. There is no barrier to doing this right now.

Technical explanation of services - YOU CAN SKIP THIS PART

This is a breakdown of the Google specific services I am planning on using:

Service Breakdown


Cloud Run - Web application hosting
- Hosts the ordering platform interface - the actual website
- Automatically scales to handle traffic fluctuations
- Pay-per-use pricing (when there are no users, there is no charge)

Cloud SQL for PostgreSQL - Database
- Stores orders, products, inventory, and customer data
- Automated backups
- PostgreSQL is by far the leading relational database solution, widely adopted for its reliability and performance

Cloud Tasks - Background job processing
- Manages order processing, inventory updates, and scheduled tasks
- Guarantees task completion with automatic retry logic
- Ensures no jobs are lost during system failures

Cloud Functions - Event-driven automation
- Triggers automated actions (email notifications, inventory sync, reporting)
- Executes only when needed, minimizing costs

Cloud Storage - Temporary file storage
- Intermediary storage for file uploads during processing
- Supporting infrastructure, not a core platform component

Memorystore (Redis) - In-memory caching
- Provides fast, low-latency access to frequently used data (e.g., session data, product catalog, inventory counts)
- Reduces load on Cloud SQL and speeds up user-facing operations

CDN: Cloudflare R2 for Product Image Delivery

Use

To serve product images to our website visitors. Every time someone views a product, we deliver web optimized versions of the images. We also maintain version history of these images.

  • Images will be available for all future JAMP projects (ex: Back in stock emails)
  • THIS IS ONLY FOR PUBLIC FACING ASSETS

Why Cloudflare R2

Monthly Cost Breakdown:

The bulk of our image uploads will be done during the initial setup, then 2-3 images per new product as they are launched. Image uploads are independent of website views, you can be in the Light scenario for image uploads but Heavy scenario for website views in a given month.

Note: Google has $18/mo minimum, Azure has $35/mo minimum, Cloudflare has no minimum

Cloudflare is 96-99% cheaper because:

  1. No bandwidth fees (competitors charge heavily for data transfer)
  2. No monthly minimums (Google/Azure have $18-35 minimums)
  3. Cloudflare uses R2 as a loss leader to attract customers for their security products (which we don't need)

Trade-off

Think of DNS as the internet's phone book. When you type "jamppharma.com" into a browser, DNS is what translates that name into the actual IP address where the website is hosted.

Cloudflare can read the active DNS settings from our current provider and copy them over. We just update a setting to say "Cloudflare is managing this now".

DNS is a "set it and forget it" service. Most companies don't touch it for years after setting it up.

Recommendation

Use Cloudflare R2. The savings are substantial in the long-term and the service is reliable. Cloudflare powers ~40% of internet traffic among the top 1,000 most visited websites globally. This is the only case where a service is so much better than the Google alternative that it's worth the tradeoff.
